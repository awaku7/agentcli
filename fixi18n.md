# i18n 修正課題

## 1. 本体翻訳（.po）のコンパイルエラー

新規追加した4言語（he, ro, hu, el）の `msgfmt` コンパイルが通らない。

### エラー種別

#### A. 末尾/先頭の `\n` 不一致（各言語 23件）
tr.po をテンプレートに msgstr のみ書き換えているが、tr.po の msgstr が持つ末尾の `\n` が翻訳で失われている。

- 例: msgid が `"...\n"` で終わる場合、msgstr も `"...\n"` で終わる必要がある
- 影響行: 74, 294, 300, 308, 316, 320, 326, 334, 338, 346, 393, 402, 416, 426, 436, 445, 458, 465, 762, 1785, 1789, 2046（末尾）, 2093（先頭）
- **修正方法**: tr.po の msgstr が `\n` で終わる/始まる場合、translate_safe() で翻訳後も同じ改行を付与する

#### B. プレースホルダー欠落（言語により件数・内容が異なる）

| 言語 | エントリ | 欠落したプレースホルダー |
|------|---------|------------------------|
| ro   | line 250 | `%(total)s` が文字化け |
| ro   | line 1814 | `%(default)s` が欠落 |
| hu   | line 250 | `%(total)s` が欠落 |
| hu   | line 1814 | `%(default)s` が文字化け |
| el   | line 782 | `%(name)s` が欠落 |
| el   | line 1814 | `%(default)s` が欠落 |
| he   | (なし) | 問題なし |

- **修正方法**: 該当エントリの msgid からプレースホルダーを抽出し、msgstr にも存在することを確認。欠けている場合は手動で追加する。

### 修正手順

1. tr.po をパースし、各エントリの msgstr の先頭/末尾の `\n` 有無を記録
2. 翻訳時に、プレースホルダーをトークンで保護（`%(name)s` → `__PH0__`）
3. 翻訳後、トークンを戻し、tr.po と同じ改行を付与
4. `msgfmt -c` で検証

## 2. Tool JSON 翻訳の追加

`md2idx_tool.json` には現在30言語あるが、他のツールJSON（例: `a2a_poll_tool.json`）も同様の30言語セットを持っている。

以下の5言語が未追加のツールJSONがある場合に追加が必要:

| コード | 言語 |
|--------|------|
| tr     | トルコ語 |
| he     | ヘブライ語 |
| ro     | ルーマニア語 |
| hu     | ハンガリー語 |
| el     | ギリシャ語 |

### 影響を受けるツールJSON

要確認。現在 `md2idx_tool.json` は30言語対応済み。他のツール（`read_file_tool.json`, `search_files_tool.json`, `python_exec_tool.json` など）も同様に対応が必要かどうか調査する。

### 翻訳方法

各ツールJSONの `en` エントリをベースに、Google非公式 API で翻訳して追加。
