## ทำไมต้องอั๊ก?

**หลุดพ้นจากการล็อคอินของผู้ขาย** ผู้ช่วย AI ส่วนใหญ่เชื่อมโยงคุณกับผู้ให้บริการเฉพาะหรือบริการคลาวด์ uag แตกต่าง

- **ทำงานภายในเครื่อง** บนเครื่องของคุณ ข้อมูลของคุณจะอยู่กับคุณ (ยกเว้นการเรียก API ที่คุณทำ)
- **เสรีภาพของผู้ให้บริการ**: OpenAI, Claude, Gemini, DeepSeek, Ollama, Azure, Bedrock, HuggingFace... ผู้ให้บริการมากกว่า 15 ราย ทั้งหมดนี้สามารถเข้าถึงได้จากอินเทอร์เฟซเดียว สลับระหว่างตัวแปรเหล่านี้โดยกำหนดค่าตัวแปรสภาพแวดล้อมใหม่ ไม่ต้องติดตั้งใหม่ ไม่ต้องย้ายข้อมูล
- **เครื่องมือ 131 รายการ**: ไฟล์ I/O, การค้นหาเว็บ, การสร้างภาพ, Gmail, การสแกนอุปกรณ์ BLE, การบูรณาการเซิร์ฟเวอร์ MCP — **76 รายการเป็นแบบปลอดภัยแบบขนาน** (สูงสุด 8 รายการดำเนินการพร้อมกันผ่านกลุ่มเธรด กำหนดค่าได้ผ่าน `UAGENT_PARALLEL_WORKERS`) เมื่อ LLM เรียกใช้เครื่องมือหลายรายการพร้อมกัน uag จะขนานการเรียกเครื่องมือเหล่านั้นโดยอัตโนมัติ
- **3 UI + A2A**: CLI, GUI, เว็บ และโปรโตคอล Agent-to-Agent เครื่องยนต์เดียวกัน อินเทอร์เฟซใดก็ได้
- **พร้อม IoT**: SwitchBot, ECHONET Lite, Matter, UPnP — ควบคุมอุปกรณ์ในบ้านของคุณผ่าน AI
- **ทักษะตัวแทน**: ติดตั้งทักษะที่สร้างโดยชุมชนจากตลาดกลาง ขยาย uag อย่างไม่มีที่สิ้นสุด

uag คือ **ผู้ช่วย AI ของคุณตามเงื่อนไขของคุณ** ไม่เชื่อมโยงกับผู้ให้บริการ, ไม่เชื่อมโยงกับอินเทอร์เฟซ, ไม่เชื่อมโยงกับแพลตฟอร์ม

## คุณสมบัติ

### 🧠 สถาปัตยกรรมหลายผู้ให้บริการ

OpenAI / Azure / Bedrock / OpenRouter / Ollama / Gemini / Vertex AI / Claude / Grok / NVIDIA / DeepSeek / Z.AI (Zhipu AI) / HuggingFace / Alibaba Cloud (Qwen) / KIMI (Moonshot AI) / Xiaomi MiMo / LM Studio / MiniMax / **Sakana AI (Fugu)**

ผู้ให้บริการทั้งหมดใช้ชุดเครื่องมือและอินเทอร์เฟซเดียวกัน สลับโดยการตั้งค่า `UAGENT_PROVIDER` — ไม่มีการเปลี่ยนแปลงโค้ด ไม่มีการติดตั้งแยกต่างหาก

### ⚡ การดำเนินการเครื่องมือแบบขนาน

เมื่อ LLM ร้องขอเครื่องมือหลายรายการพร้อมกัน uag **จะขนานกัน** โดยอัตโนมัติ
เครื่องมือ 76 รายการถูกทำเครื่องหมายเป็น `x_parallel_safe` และดำเนินการพร้อมกันผ่าน `ThreadPoolExecutor` (8 เธรดตามค่าเริ่มต้น ตั้งค่า `UAGENT_PARALLEL_WORKERS` เพื่อเปลี่ยนแปลง)

**ตัวอย่าง**: ถาม "ตรวจสอบสภาพอากาศในเมืองหลวงของนอร์ดิก" → LLM เรียกใช้ `search_web` × 5 ประเทศ → การค้นหาทั้ง 5 รายการทำงานแบบขนาน → ผลลัพธ์ที่รวบรวมไว้ในชุดเดียว

เครื่องมือแบบอ่านอย่างเดียว (การค้นหาไฟล์ การคำนวณแฮช รายการไดเร็กทอรี การแปล การสืบค้น DB ฯลฯ) ได้รับการขนานกันอย่างมาก

### 🔄 ความต่อเนื่องของเซสชัน

- **Switch providers mid-session** with `UAGENT_PROVIDER` — conversation history is preserved.
- **Reload past sessions** with `:load <index>` — pick up where you left off.
- **Tool result caching** avoids redundant re-execution when the same tool call repeats.

### 🤖 นักบินอัตโนมัติ (`:auto`)

uag สามารถ **ติดตามเป้าหมายโดยอัตโนมัติผ่าน LLM หลายรอบ** เหมาะสำหรับงานที่ซับซ้อนหลายขั้นตอนที่ต้องการการปรับแต่งซ้ำๆ

- **วิธีการทำงาน**: แต่ละรอบมีคำถามหลัก (ขั้นตอน A) ตามด้วยการตัดสินของผู้ตรวจสอบ (ขั้นตอน B) ที่ตัดสินใจว่า "เสร็จสมบูรณ์หรือดำเนินการต่อ"
- **ผู้ให้บริการรายเดียวกัน API เดียวกัน**: การตัดสินของผู้ตรวจสอบใช้เส้นทางโค้ดที่เหมือนกันเป็นแบบสอบถามหลัก รวมถึงการสนับสนุน Responses API
- **แยกผู้พิพากษา LLM** (ไม่บังคับ): ตั้งค่า `UAGENT_AP_PROVIDER` เพื่อใช้ผู้ให้บริการ/รุ่นอื่นสำหรับผู้ตรวจสอบ (เช่น ใช้รุ่นที่ถูกกว่าในการตัดสิน)
- **ออกเมื่อใดก็ได้**: กดปุ่ม `x` เพื่อหยุดทันที แม้จะตอบกลับกลางคันก็ตาม หรือให้ผู้วิจารณ์ตัดสินใจว่าจะบรรลุเป้าหมายเมื่อใด
- **กำหนดค่าได้**: `--max-rounds N` เพื่อควบคุมงบประมาณ

ดู [README_AUTO.md](https://github.com/awaku7/agentcli/blob/main/README_AUTO.md) สำหรับเอกสารฉบับเต็ม

### 🧩 ผู้จัดการกลุ่มรัฐ

uag สามารถติดตามความคืบหน้าของงานหลายไฟล์ที่ใช้เวลานาน เมื่อ LLM ประมวลผลไฟล์หลายสิบไฟล์ `batch_state` จะยังคงอยู่ในรายการไฟล์ที่รอดำเนินการ เสร็จสมบูรณ์ และล้มเหลวลงดิสก์ หากเซสชั่นสิ้นสุดลงหรือหมดเวลา การวิ่งครั้งถัดไปจะดำเนินการต่อจากจุดที่หยุด ไม่มีอะไรสูญหาย

### 🛡 มนุษย์ในวงโคจร

`human_ask` ให้ LLM หยุดชั่วคราวและขอการยืนยันก่อนดำเนินการทำลายล้าง (การลบไฟล์ เขียนทับ คำสั่งเชลล์) คุณอยู่ในการควบคุม

### 🛑 ขัดจังหวะ (ปุ่ม c / หยุด)

หยุดการสร้างการตอบสนอง LLM ได้ตลอดเวลา และแทรกคำสั่งหยุดกลับไปยัง LLM

| อินเตอร์เฟซ | วิธีการขัดจังหวะ |
|---|---|
| **คลี** | กดปุ่ม `c` ระหว่างการสตรีม LLM — การตอบกลับปัจจุบันจะหยุดลง และ `"หยุด"` จะถูกส่งเป็นข้อความผู้ใช้ ดังนั้น LLM จะตอบกลับตามนั้น |
| **เว็บ UI** | คลิกปุ่มสีแดง **■ หยุด** (ปรากฏขึ้นโดยอัตโนมัติระหว่างการประมวลผล LLM) |
| ** GUI เดสก์ท็อป ** | คลิกปุ่มสีแดง **■** (ปรากฏขึ้นโดยอัตโนมัติระหว่างการประมวลผล LLM) |

การขัดจังหวะทำงานเหมือน "การแทรกพร้อมท์": แทนที่จะเพียงแค่ยกเลิก การขัดจังหวะจะป้อน `"หยุด"` กลับไปยัง LLM เป็นข้อความผู้ใช้ เพื่อให้สามารถสรุปหรือรับทราบการขัดจังหวะได้อย่างสวยงาม

กดปุ่ม `x` เพื่อออกจากโหมดนำร่องอัตโนมัติ (ดู [README_AUTO.md](https://github.com/awaku7/agentcli/blob/main/README_AUTO.md))

### 🕵️ ระบบอัตโนมัติของเบราว์เซอร์ & ตัวตรวจสอบเว็บ

เครื่องมือที่ใช้นักเขียนบทละครเสริมสองเครื่องมือ:

- **browser_playwright**: ทำให้เซสชันเบราว์เซอร์จริงเป็นอัตโนมัติ — นำทาง คลิก กรอกแบบฟอร์ม แยกข้อมูล จัดการโฟลว์หลายหน้า ทำงานแบบไม่มีหัวหรือมีศีรษะ
- **playwright_inspector**: บันทึกการเปลี่ยนผ่านเบราว์เซอร์ จับภาพสแนปชอต DOM และภาพหน้าจอในแต่ละขั้นตอน มีประโยชน์สำหรับการแก้ไขการโต้ตอบบนเว็บหรือตรวจสอบการเปลี่ยนแปลงเพจเมื่อเวลาผ่านไป

### 🔄 กำลังโหลดเครื่องมือแบบไดนามิก

`tool_catalog` และ `tool_load` ช่วยให้คุณค้นพบและเปิดใช้งานเครื่องมือในขณะรันไทม์

## ปรัชญาโครงการ

uag ปรารถนาที่จะเป็น **AI ของคุณ บนเครื่องของคุณ ตามเงื่อนไขของคุณ**

- ไม่มีการพึ่งพา SaaS — ทำงานภายในเครื่อง
- ไม่มีการล็อคอินของผู้ให้บริการ — สลับได้ตลอดเวลา
- ไม่มีการล็อคอิน UI — CLI / GUI / Web / A2A
- ไม่มีการล็อคอินฟีเจอร์ — ขยายขอบเขตด้วยเครื่องมือและทักษะ

ประสบการณ์ตัวแทน AI ฟรี ไม่มีการล็อคอินจากผู้จำหน่าย
