## Почему уаг?

**Освободитесь от привязки к поставщику.** Большинство ИИ-помощников привязывают вас к конкретному поставщику или облачному сервису. уаг другой.

- **Запускается локально** на вашем компьютере. Ваши данные остаются с вами (за исключением вызовов API, которые вы совершаете).
- **Свобода поставщиков**: OpenAI, Claude, Gemini, DeepSeek, Ollama, Azure, Bedrock, HuggingFace... Более 15 поставщиков, все доступны из единого интерфейса. Переключайтесь между ними путем перенастройки переменных среды — без переустановки и миграции.
- **131 инструмент**: файловый ввод-вывод, веб-поиск, генерация изображений, Gmail, сканирование устройств BLE, интеграция с сервером MCP — **76 из них безопасны для параллельного выполнения** (до 8 выполняются одновременно через пул потоков, настраивается через `UAGENT_PARALLEL_WORKERS`). Когда LLM запускает несколько вызовов инструментов одновременно, uag автоматически распараллеливает их.
- **3 интерфейса + A2A**: интерфейс командной строки, графический интерфейс, веб-интерфейс и протокол взаимодействия агентов. Тот же движок, любой интерфейс.
- **Подготовка к Интернету вещей**: SwitchBot, ECHONET Lite, Matter, UPnP — управляйте своими домашними устройствами с помощью искусственного интеллекта.
- **Навыки агента**: установите навыки, созданные сообществом, с торговой площадки. Расширяйте uag бесконечно.

uag — **ваш помощник по искусственному интеллекту на ваших условиях**. Не привязан к провайдеру, не привязан к интерфейсу, не привязан к платформе.

## Функции

### 🧠 Многопровайдерная архитектура

OpenAI / Azure / Bedrock / OpenRouter / Ollama / Gemini / Vertex AI / Claude / Grok / NVIDIA / DeepSeek / Z.AI (Zhipu AI) / HuggingFace / Alibaba Cloud (Qwen) / KIMI (Moonshot AI) / Xiaomi MiMo / LM Studio / MiniMax / **Sakana AI (Fugu)**

Все поставщики используют один и тот же набор инструментов и интерфейс. Переключитесь, установив `UAGENT_PROVIDER` — никаких изменений кода, никаких отдельных установок.

### ⚡ Параллельное выполнение инструментов

Когда LLM запрашивает несколько инструментов одновременно, uag **автоматически распараллеливает** их.
76 инструментов помечены `x_parallel_safe` и выполняются одновременно через `ThreadPoolExecutor` (8 потоков по умолчанию; установите `UAGENT_PARALLEL_WORKERS` для изменения).

**Пример**: запрос «Узнайте погоду в столицах Северных стран» → LLM запускает `search_web` × 5 стран → все 5 поисковых запросов выполняются параллельно → результаты собираются в один пакет.

Инструменты только для чтения (поиск файлов, вычисление хеша, просмотр каталогов, перевод, запросы к базе данных и т. д.) активно распараллеливаются.

### 🔄 Непрерывность сеанса

- **Switch providers mid-session** with `UAGENT_PROVIDER` — conversation history is preserved.
- **Reload past sessions** with `:load <index>` — pick up where you left off.
- **Tool result caching** avoids redundant re-execution when the same tool call repeats.

### 🤖 Автопилот (`:auto`)

uag может **автономно добиваться цели на протяжении нескольких раундов LLM**. Идеально подходит для сложных, многоэтапных задач, требующих итеративной доработки.

- **Как это работает**: каждый раунд состоит из основного запроса (шаг А), за которым следует заключение рецензента (шаг Б), которое принимает решение «ЗАВЕРШИТЬ или ПРОДОЛЖИТЬ?»
- **Тот же поставщик, тот же API**: по мнению рецензента, используется тот же путь к коду, что и в основном запросе, включая поддержку API ответов.
- **Отдельное судейство LLM** (необязательно): установите `UAGENT_AP_PROVIDER`, чтобы использовать другого поставщика/модель для рецензента (например, использовать более дешевую модель для судейства).
- **Выйти в любое время**: нажмите клавишу «x», чтобы немедленно остановиться, даже в середине ответа. Или позвольте рецензенту решить, когда цель будет достигнута.
- **Настраивается**: `--max-rounds N` для контроля бюджета.

Полную документацию см. в [README_AUTO.md](https://github.com/awaku7/agentcli/blob/main/README_AUTO.md).

### 🧩 Пакетный менеджер состояний

uag может отслеживать ход выполнения длительных многофайловых задач. Когда LLM обрабатывает десятки файлов, `batch_state` сохраняет на диске список ожидающих, завершенных и неудавшихся файлов. Если сессия заканчивается или время раунда истекает, следующий запуск возобновляется с того места, где он был остановлен — ничего не теряется.

### 🛡 Человек в курсе

Human_ask позволяет LLM приостанавливать работу и запрашивать ваше подтверждение перед выполнением разрушительных операций (удаление файлов, перезапись, команды оболочки). Вы остаетесь под контролем.

### 🛑 Прерывание (клавиша c/кнопка «Стоп»)

Остановите генерацию ответа LLM в любой момент и подайте команду остановки обратно в LLM.

| Интерфейс | Как прервать |
|---|---|
| **CLI** | Нажмите клавишу `c` во время потоковой передачи LLM — текущий ответ останавливается, и `"Стоп"` отправляется как пользовательское сообщение, поэтому LLM отвечает соответствующим образом |
| **ВЕБ-интерфейс** | Нажмите красную кнопку **■ Стоп** (появляется автоматически во время обработки LLM) |
| **Графический интерфейс рабочего стола** | Нажмите красную кнопку ***** (появляется автоматически во время обработки LLM) |

Прерывание работает как «быстрая инъекция»: вместо простого прерывания оно передает «Стоп» обратно в LLM в виде пользовательского сообщения, позволяя ему корректно завершить или подтвердить прерывание.

Нажмите клавишу «x», чтобы выйти из режима автопилота (см. [README_AUTO.md](https://github.com/awaku7/agentcli/blob/main/README_AUTO.md)).

### 🕵️ Автоматизация браузера и веб-инспектор

Два взаимодополняющих инструмента, основанных на драматурге:

- **browser_playwright**: автоматизируйте реальные сеансы браузера — перемещайтесь, щелкайте, заполняйте формы, извлекайте данные, обрабатывайте многостраничные потоки. Работает без головы или с головой.
- **playwright_inspector**: записывайте переходы браузера, делайте снимки DOM и снимки экрана на каждом этапе. Полезно для отладки веб-взаимодействий или аудита изменений страниц с течением времени.

### 🔄 Динамическая загрузка инструментов

«tool_catalog» и «tool_load» позволяют обнаруживать и включать инструменты во время выполнения.
Не нужно загружать все при запуске — активируйте только то, что вам нужно, и тогда, когда вам это нужно.

## Философия проекта

uag стремится стать **вашим ИИ, на вашей машине, на ваших условиях**.

- Нет зависимости от SaaS — работает локально.
- Нет привязки к провайдеру — переключайтесь в любое время
- Нет привязки пользовательского интерфейса — CLI/GUI/Web/A2A
- Нет привязки к функциям — расширяйте возможности с помощью инструментов и навыков.

Бесплатный AI-агент, свободный от привязки к поставщику.
