# Discord でできること

`src/uagent/tools/discord_channel_tool.py` を使うと、Discord の特定チャンネルと対話できます。

## できること

- チャンネルへメッセージ送信
- 送信後に返信を待つ
- チャンネルの最新メッセージ取得
- 特定チャンネルを会話の窓口にする
- 通知専用の受け口にする
- 簡易コマンドの受付に使う
- ボット経由の会話中継
- FAQ っぽい自動応答
- 監視チャンネルの自動通知
- 返信や発言のログ収集
- 外部 API 連携の入口
- 定期投稿やリマインド
- スレッド運用の補助
- DM 対応
- 添付ファイルの保存/転送
- 既読っぽい運用補助
- ロール別の応答切り替え
- ルールベースの自動振り分け
- 自動リマインド
- 複数チャンネル巡回
- 通知のまとめ送信
- 会話ログの保存
- Webhook との使い分け
- 申請/受付フローの補助
- キーワード監視
- 定型文の自動返信
- 返信候補の生成
- 障害連絡やアラートの中継
- 相談窓口の一次対応

## 使うもの

- Bot token
- `DISCORD_BOT_TOKEN` 環境変数
- 対象チャンネル ID

## Bot token の取得方法（PC ベース）

1. PC のブラウザで Discord 開発者ポータルを開く
   - https://discord.com/developers/applications
2. Discord アカウントでログインする
3. 右上または一覧から `New Application` を押す
4. アプリ名を入力して作成する
5. 左メニューの `Bot` を開く
6. `Add Bot` を押して Bot を作成する
7. 表示された Bot 画面で `Reset Token` または `Token` 付近からトークンを取得する
8. トークンをコピーして安全な場所に保存する
9. 必要なら `.env` に次のように設定する

```env
DISCORD_BOT_TOKEN=your_bot_token_here
```

## Bot をサーバーに参加させる

1. 左メニューの `OAuth2` → `URL Generator` を開く
2. `bot` を選ぶ
3. 必要な権限を選ぶ
4. 生成された URL を開く
5. 招待したいサーバーを選んで追加する

## チャンネル ID の確認

1. Discord のユーザー設定を開く
2. `詳細設定` の `開発者モード` を ON にする
3. 対象チャンネルを右クリックする
4. `ID をコピー` を選ぶ

## tool 名

- `discord_channel_chat`

## action

### `send`
メッセージを送信して終了します。

### `send_and_wait`
メッセージを送信したあと、一定時間返信を待ちます。
- `wait_s`: 待機秒数
- `poll_interval_s`: 監視間隔
- `exclude_bots`: Bot を除外するか

### `history`
チャンネルの最新メッセージを取得します。
- `limit`: 取得件数

## 例

### 送信
```json
{
  "action": "send",
  "channel_id": "123456789012345678",
  "message": "こんにちは"
}
```

### 送信して返信待ち
```json
{
  "action": "send_and_wait",
  "channel_id": "123456789012345678",
  "message": "今の状態を教えて",
  "wait_s": 30,
  "poll_interval_s": 2,
  "exclude_bots": true
}
```

### 履歴取得
```json
{
  "action": "history",
  "channel_id": "123456789012345678",
  "limit": 10
}
```

## 注意

- Discord 側の権限が必要です
- Bot は対象チャンネルに参加している必要があります
- 2000 文字を超える送信はできません
- 送信後の待機は REST polling 方式です
- ループやスパムにならないよう注意してください
- トークンは `DISCORD_BOT_TOKEN` に入れてください
