## 方針: index 化の高速化

### 基本方針
- まずは埋め込み API の往復回数を減らす。
- チャンク単位の逐次処理は避け、まとめて送る。
- SQLite 書き込みはなるべくまとめる。
- 差分がないファイルは再計算しない。

### 実装目安
- 1 リクエストあたり 16 個前後のチャンクをまとめる。
- チャンクが長い場合は 8 個、短い場合は 32 個まで調整する。
- 件数固定ではなく、合計文字数の上限も設ける。
- 目安は合計 12,000〜20,000 文字程度。

### 調整対象
- `chunk_size` は大きめにする。
- `overlap` は必要最小限にする。
- `executemany()` で一括保存する。
- ファイル更新判定は `mtime` だけでなく、必要ならサイズやハッシュも使う。
- 埋め込み設定ごとに別 DB を使う。
