# 主要なアップデート（直近30日間）

このドキュメントは、直近30日間に `uag` プロジェクトに導入された主要なアップデート、アーキテクチャの強化、および新機能についてまとめたものです。

---

## 1. 専門サブエージェント・アーキテクチャ (`run_sub_agent`)
親エージェント（オーケストレーター）の制御下で、特定の専門タスクを安全かつサンドボックス化された環境で実行する「サブエージェント実行フレームワーク」が導入されました。

* **役割ベースのサブエージェント**:
  * `planner` (計画設計): 段階的な実行計画の策定、リスクの特定、制約事項の整理。
  * `reviewer` (コード監査): コード、設計、パッチ案のバグ、セキュリティ脆弱性、無限ループリスクの監査。
  * `summarizer` (文脈圧縮): 長い会話履歴やログを必要不可欠なコンテキストに要約・圧縮。
  * `patch_designer` (パッチ設計): 実際の書き換えを行わずに、安全で具体的なコード置換パッチ（差分）を設計。
  * `error_analyst` (エラー解析): コンパイル、テスト、または実行時エラーを解析し、根本原因（Root Cause）を特定。
* **機能（役割）別の個別上書き (オーバーライド)**:
  * サブエージェントの役割ごとに、LLMプロバイダ、モデル名、APIキーを個別に上書きできる環境変数（例: `UAGENT_SUB_AGENT_SUMMARIZER_PROVIDER=gemini` を指定して、要約タスクのみを高速・安価な Gemini 2.5 Flash に処理させる）をサポートしました。
* **多言語対応 (I18N)**:
  * サブエージェントツールの仕様および説明文を、世界30カ国語に完全ローカライズしました。

---

## 2. Python 3.11+ へのモダン化
コードベース全体を Python 3.11+ にモダン化し、型安全性、パフォーマンス、およびコードの保守性を大幅に向上させました。

* **モダンな型ヒント構文**:
  * `typing.List` や `typing.Dict` の代わりに Python 3.11 標準の `list` や `dict` を使用し、`typing.Union`/`typing.Optional` の代わりに `|` 演算子を使用するモダンな構文へ全面的に移行しました。
* **コードのクリーンアップ**:
  * 非推奨となった古いインポート、レガシーな互換レイヤー、および未使用のモジュールを完全に削除しました。
  * すべての変更について厳密な構文コンパイルチェック (`py_compile`) を実施し、動作を検証しました。

---

## 3. 決定論的推論と設定温度（Temperature）の統一
ツールの呼び出し、JSONフォーマットの出力、および論理的な思考ステップの安定性を確保するため、生成温度（`temperature`）の設定を最適化しました。

* **統一されたデフォルト温度**:
  * エージェントループ実行時、すべての主要LLMプロバイダ（Gemini, Vertex AI, Claude, OpenAI, Azure, Bedrock, OpenRouter, Grok, NVIDIA）において、デフォルトの `temperature` を **`0.2`** に統一しました。
* **プロバイダ別の個別上書き**:
  * 環境変数（例: `UAGENT_GEMINI_TEMPERATURE`, `UAGENT_CLAUDE_TEMPERATURE`, `UAGENT_OPENAI_TEMPERATURE`）を介して、プロバイダごとに温度を個別に上書きできる仕組みを導入しました。

---

## 4. 多言語ツールカタログと検索ランキングの改善
多言語環境におけるツールの発見を容易にし、検索の精度を向上させるための改善を行いました。

* **トークン化と検索精度の向上**:
  * ツールカタログの多言語トークン化サポートを追加しました。
  * ユーザーの意図に基づいて、関連性の高いツールをより正確に上位表示する検索ランキングアルゴリズムを改善しました。
* **対応言語の拡大**:
  * ベンガル語 (`bn`)、ペルシャ語 (`fa`)、モンゴル語 (`mn`)、マラーティー語 (`mr`) の翻訳リソースを新たに追加し、グローバル対応をさらに拡大しました。

---

## 5. バッチ状態管理と堅牢なファイル操作
大規模な複数ファイルのバッチ処理を安全かつ効率的に進めるための機能を強化しました。

* **バッチ状態の追跡**:
  * 処理の進捗管理、中断されたタスクの再開、ファイルごとの実行結果の記録を行う堅牢なバッチ状態追跡ツール（`batch_state`）を導入しました。
* **ファイル操作の改善**:
  * `replace_in_file` ツールのパフォーマンス向上と、改行コードに関する診断機能を強化しました。
  * Windows環境におけるシェルコマンドのクォーティングおよび引数のエスケープ処理を堅牢化しました。

---

## 6. GUI とログ描画の強化
グラフィカルインターフェースにおけるログの視認性を向上させ、ユーザー体験を改善しました。

* **ANSI カラー描画**:
  * ターミナルのカラーログをGUI上で正しく解析・表示するための ANSI HTML 描画ヘルパーを追加しました。
  * GUIコンソールにおけるログのカラー描画の不具合を修正しました。
