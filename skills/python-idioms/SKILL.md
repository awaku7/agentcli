## name: python-idioms description: | Pythonを書く/レビューする時に「よりPythonらしい書き方」を素早く参照するための定石集（自分用の実務チートシート）。 license: Apache-2.0

# 目的

- Pythonコードを「動く」から「読みやすい・壊れにくい・直しやすい」へ寄せるための参照メモ。
- 実装中/レビュー中に、迷いがちな書き方の判断基準を短時間で引けるようにする。

# 使い方

- 実装中: `references/TECHNIQUES.md` から該当トピック（例: f-string / pathlib / 例外）を引いて適用する。
- レビュー時: `references/CHECKLIST.md` を上から順に当てて、改善点を列挙する。

# 収録（参照ファイル）

- `references/TECHNIQUES.md`
  - テクニック（イディオム）別の「なぜ/いつ/例/注意点」
- `references/CHECKLIST.md`
  - 実務レビュー用チェックリスト
- `references/SOURCES.md`
  - 参考リンク（外部記事など）

# このスキルの方針

- 外部記事の内容を丸写しせず、要点・判断基準・自分の実務観点に再整理して書く。
- 「必ずこうする」ではなく、トレードオフ（可読性/安全性/性能/依存）も併記する。
